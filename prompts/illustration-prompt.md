任务:为下面这篇文章生成一张**扁平卡通风格的解释图**(flat cartoon explanation illustration),让没读过文章的人也能一眼看懂文章的核心结构。

**画幅比例:由内容决定,不限定。** 如果文章是双行对照(如「AI 三层 vs 编程三层」),用 3:2 或 4:3 横版,放得下两行。如果是流程递进,可以用横长条(2.45:1 / 3:1)。如果是层级深度,可以用竖版(3:4 / 4:5)。如果是凝聚式概念,方形(1:1)也行。**比例服务于内容能否被一眼读懂,不服从任何外部规则。**

## 风格规范

- **整体感**:像 New Yorker 杂志插图、xkcd、或者高级科普读物的插画——既有趣又有思想深度
- **造型**:几何化简化人物(火柴人 / 圆头方身),线条清晰,无写实细节
- **配色**:温暖,克制,最多 4–5 种主色。建议:米白底 + 深色线条 + 1–2 个强调色(橙红、墨绿、深蓝任选)。绝不要饱和霓虹、廉价渐变、3D 拟真
- **质感**:纯平面 / 轻微纸张颗粒 / 手绘线条感都可。不要金属、玻璃、3D 光泽
- **气质**:像在白纸上手绘的概念图,不像 PPT 或 Canva 模板

## 构图要求

- **比例由内容决定**(见上),不强求横长条
- **多个分区并列**:把文章的核心层级 / 阶段 / 对比关系并列展开,从左到右、上下分层、或环形排布,以最易读为准
- **箭头 / 台阶 / 流程线**:用通用的视觉符号连接各区,表达递进、层级或对比关系
- **每区一个核心场景**:不要堆细节,每个分区只画 1 个主场景 + 1 个核心物件
- **中文短标签**:每个分区可以有 1 个 2–6 字的中文标签注释。标签字必须准确、可读、无伪汉字
- **空白克制**:分区之间留呼吸空间,整体不能挤

## 视觉元素优先级

第一层级:核心场景 / 人物 / 物件(每分区一个)
第二层级:连接元素(箭头、台阶、虚线、流程线)
第三层级:中文标签
第四层级:背景(留白或极淡纹理)

## 必须避免

- 真人脸部(用简化几何代替)
- 文字过多(只用关键标签,不是 PPT)
- 抽象到看不懂(必须能"读图理解文章")
- 风格不统一(不同分区用同一种线条粗细和配色)
- 廉价插画感、儿童读物感、中国风滥用
- 任何水印、签名、Logo、二维码、英文版权说明
- 错字、漏字、伪中文笔画
